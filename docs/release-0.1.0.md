## What changed vs. the smoke config

- **NSIS target** (was `dir`): assisted wizard, `perMachine: false` + `allowElevation: false`
  (no admin), `allowToChangeInstallationDirectory: true`, desktop + Start-menu shortcuts.
- **Icon**: `assets/icon.png` (256×256, generated by `scripts/make-icon.mjs` — pure Node, no
  rasterizer/GPU dependency), from which electron-builder derives the app/exe/installer `.ico`.
- **Version/metadata**: productName `Jarvis`, appId `com.jarvis.assistant`, version `0.1.0`
  (from `package.json`), copyright wired.
- **Size pruning**: `!**/*.d.ts` / `!*.js.map` / `!*.md` excludes. googleapis alone ships ~164 MB
  of `.d.ts` (of ~184 MB across all deps); these never load at runtime, so dropping them is safe
  and touches no module resolution. (googleapis' `build/src/index.js` eagerly `require`s **every**
  api module, so pruning individual api dirs is NOT safe — only the `.d.ts` are.)
- **In-app docs**: `docs/wakeword-setup.md` shipped as an extraResource; the settings "how to
  train a wake word" link now resolves in the installed app (windows.ts routes it to the shipped
  file via the shell, and hardens all renderer navigation: external links open in the browser,
  in-window navigation is blocked).

## Autostart

`app.setLoginItemSettings({ openAtLogin, args: ['--hidden'] })` registers the per-user HKCU Run
value `Jarvis` pointing at the installed exe; `--hidden` makes login launches go straight to the
tray. The installer never creates the value (the app does, at runtime, when the user enables it);
the custom uninstall macro (`packages/app/build/installer.nsh`) deletes it so no orphan autostart
entry survives uninstall.

## Uninstall data policy (scrub vs. keep)

`deleteAppDataOnUninstall: false` — `%APPDATA%/Jarvis` (sessions, second-brain vault + index,
plugin config/secrets, fetched `models/` ~900 MB) is **kept**. Sessions and the brain are genuine
user content; models are expensive to re-fetch. Full scrub = delete the folder manually.

## Dual-ABI native module strategy (better-sqlite3)

One `node_modules` copy, three runtimes:

- **Resting/tests** = Node ABI. `npm install`, `npm test`, `npm run dist`, `npm run package` all
  end at the Node ABI (dist/package rebuild a private Electron-ABI copy at package time and restore
  Node after). `npm test` therefore always works with no manual step.
- **Dev Electron + brain** = Electron ABI: `npm run rebuild:electron` before `npm run dev`, then
  `npm run rebuild:node` before tests again. (Voice-only dev doesn't load sqlite.)
- **Packaged app** = self-contained Electron-ABI copy built by electron-builder's `@electron/rebuild`.

## Deferred-item dispositions (cdd/plan/amendments.md)

- **ONNX embedder warm-up** — DONE. `OnnxEmbedder.warmUp()` + `BrainRuntime.warmUp()`;
  `src/main/index.ts` fires it (fire-and-forget) at startup when `secondBrain.enabled`, so the
  first recall/capture turn doesn't pay the ONNX session-create cost. Logs
  `[main] brain embedder warmed up`.
- **Packaged wakeword-setup.md link** — DONE (shipped extraResource + shell open; see above).
- **Renderer `sandbox: true`** — attempted, left **off** with a precise code comment
  (`src/main/windows.ts`). electron-vite emits the preload as ESM `index.mjs` (forced by the
  package's `"type":"module"`), and Electron's sandbox loads preloads through a CommonJS-only path —
  a sandboxed ESM preload fails to load, killing `window.jarvis` and the whole IPC bridge.
  Converting the preload to a CJS artifact under an ESM package is non-trivial and unverifiable
  headless, so with `contextIsolation:true` + `nodeIntegration:false` already providing the real
  boundary, sandbox stays off pending a CJS preload build.
